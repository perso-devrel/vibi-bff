# 코드 리뷰 스킬

사용자가 "리뷰"라는 단어를 언급하면 자동으로 실행한다.

## 리뷰 대상 파악

1. `git diff`로 변경된 파일 확인 (staged + unstaged)
2. 변경이 없으면 `git diff HEAD~1`로 마지막 커밋 변경사항 확인
3. 사용자가 특정 파일을 지정했으면 해당 파일만 리뷰

## 리뷰 기준

아래 4가지 관점에서 검사한다. 문제가 없는 항목은 언급하지 않는다.

### 1. 중복 제거
- 같은 로직이 2곳 이상 반복되는지
- 공통 유틸/헬퍼로 추출할 수 있는 패턴이 있는지
- 비슷한 쿼리, 검증 로직, 에러 처리가 복사된 곳

### 2. 효율성
- 불필요한 DB 호출 (N+1 쿼리, 미사용 select 필드)
- 비효율적인 반복문, 불필요한 await, 병렬 처리 가능한 순차 호출
- 메모리 낭비 (큰 배열 복사, 불필요한 중간 변수)

### 3. 보안
- SQL injection, XSS, 인증/인가 우회 가능성
- 민감정보 노출 (에러 메시지에 내부 정보, 로그에 토큰)
- 입력값 검증 누락 (타입, 길이, 범위)
- IDOR (다른 사용자 리소스 접근 가능 여부)

### 4. 클린 코드
- 불명확한 변수/함수명
- 과도한 중첩 (3단계 이상 if/for)
- 미사용 import, 변수, 함수
- 매직넘버, 하드코딩된 문자열
- 일관성 없는 에러 처리 패턴

## 출력 형식

본론만 간결하게. 인사말, 요약, 칭찬 없이 바로 문제점을 나열한다.

```
### [카테고리]
- `파일:라인` — 설명
```

문제가 없으면 "이상 없음"만 출력한다.

### 심각도 표시
- **[심각]** — 즉시 수정 필요 (보안 취약점, 데이터 손실 위험)
- **[권장]** — 수정하면 좋음 (효율, 중복, 가독성)
